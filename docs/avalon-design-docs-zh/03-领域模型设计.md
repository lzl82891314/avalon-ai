# 03 - 领域模型设计

## 1. 主要聚合与实体

### 1.1 GameSession
表示一局游戏。


## 2. 受控枚举 / 受控类型

### GameStatus
- `WAITING`
- `RUNNING`
- `PAUSED`
- `ENDED`
- `RECOVERING`

### GamePhase
建议最小集合：
- `ROLE_REVEAL`
- `ROUND_START`
- `DISCUSSION`
- `TEAM_PROPOSAL`
- `TEAM_VOTE`
- `MISSION_ACTION`
- `MISSION_RESOLUTION`
- `ASSASSINATION`
- `GAME_END`
- `WAITING_FOR_HUMAN_INPUT`

### Camp
- `GOOD`
- `EVIL`

### PlayerControllerType
- `LLM`
- `HUMAN`
- `SCRIPTED`

## 3. 动作模型

所有合法玩家动作必须显式建模，并由规则引擎校验。

### 3.1 基础动作接口

```java
public interface PlayerAction {
    String type();
}
```

### 3.2 具体动作类型

```java
public class PublicSpeechAction implements PlayerAction {
    private String speechText;
}

public class TeamProposalAction implements PlayerAction {
    private List<String> selectedPlayerIds;
}

public class TeamVoteAction implements PlayerAction {
    private VoteChoice vote; // APPROVE / REJECT
}

public class MissionAction implements PlayerAction {
    private MissionChoice choice; // SUCCESS / FAIL
}

public class AssassinationAction implements PlayerAction {
    private String targetPlayerId;
}
```

## 4. 允许动作模型

系统不能只根据 phase 粗暴判断，要明确某个玩家当前允许做什么。

```java
public class AllowedActionSet {
    private String gameId;
    private String playerId;
    private List<String> actionTypes;
}
```

例如：
- 讨论阶段：`PUBLIC_SPEECH`
- 组队阶段且你是队长：`TEAM_PROPOSAL`
- 投票阶段：`TEAM_VOTE`
- 任务阶段且你在队伍里：`MISSION_ACTION`
- 刺杀阶段且你是刺客：`ASSASSINATION`

## 5. 领域服务

### 5.1 GameRuleEngine
负责：
- 动作合法性校验
- 阶段推进
- 胜负判定
- 队伍人数规则
- 失败票阈值规则

### 5.2 RoleAssignmentService
负责：
- 根据规则与模板构造角色牌组
- 按 seed 洗牌
- 分配到 seat
- 生成基础角色分配结果

### 5.3 VisibilityService
负责：
- 计算某个玩家当前可见信息
- 根据规则与角色生成私有知识
- 过滤不应暴露的字段

### 5.4 SetupValidationService
负责：
- 校验规则集与模板是否匹配
- 校验玩家数与 seat 是否匹配
- 校验角色组合是否符合规则

## 6. 玩家控制器抽象

```java
public interface PlayerController {
    PlayerActionResult act(PlayerTurnContext context);
}
```

### 设计目的
通过这个抽象统一支持：
- 纯 LLM 自动局
- 1 人类 + 多 LLM 单机局
- 多人联机局
- 脚本基准测试局

## 8. 给开发 Agent 的实现要求

### 必做
- 所有实体先做成清晰的 POJO / record / value object
- 所有状态变化走显式方法或领域服务
- 所有动作类型必须可校验、可序列化

### 禁止
- 禁止大量 stringly-typed 魔法字符串散落
- 禁止把领域逻辑放进 controller 或 prompt builder
- 禁止绕过 `PlayerController` 直接让某种玩家特殊处理
