# 08 - 实现路线图

## Phase 1：核心领域与规则基础

### 目标
先不接 LLM，先把游戏真相层做稳。

### 任务
1. 建立 Maven 多模块骨架
2. 定义规则集 / 角色 / 模板配置模型
3. 实现配置加载与校验
4. 实现基于 seed 的角色分配服务
5. 实现 5 人经典局规则引擎
6. 实现可见性服务
7. 实现脚本玩家控制器
8. 脚本玩家跑通一整局

### 验收标准
- 不接 LLM 也能完整跑一局
- 身份分配正确
- 视野正确
- 阶段推进正确

---

## Phase 2：持久化与恢复地基

### 任务
1. 建表迁移（Flyway / Liquibase）
2. 实现事件流仓储
3. 实现游戏快照
4. 实现玩家记忆快照
5. 实现回放投影
6. 实现恢复服务

### 验收标准
- 中断后可以恢复继续
- 可以按阶段查看状态变化

---

## Phase 3：API 与运行控制

### 任务
1. 创建游戏 / 启动接口
2. step / run 接口
3. 状态、事件、私有视图查询接口
4. 管理员审计接口

### 验收标准
- 仅通过 REST 就能控制与查看一局游戏

---

## Phase 4：接入 LLM 控制器

### 任务
1. 定义模型配置与 Agent 配置
2. 基于 Spring AI 实现 AgentGateway
3. 实现 PromptBuilder
4. 实现结构化响应解析
5. 实现校验与重试策略
6. 实现 LlmPlayerController

### 验收标准
- 至少一个席位可改为 LLM 玩家
- 最终 5 个席位都可由 LLM 驱动

---

## Phase 5：记忆能力完善

### 任务
1. 定义玩家记忆结构
2. 实现记忆合并策略
3. 将记忆注入 Prompt
4. 在每回合或每轮后保存快照

### 验收标准
- 中断恢复后，玩家信念与策略状态不丢失

---

## Phase 6：真人玩家模式

### 任务
1. 实现 HumanPlayerController
2. 实现等待态行为
3. 实现真人动作提交接口
4. 验证人机混合局

---

## Phase 7：回放与评测

### 任务
1. 构建回放读模型
2. 构建审计摘要查看能力
3. 构建玩家表现统计
4. 为未来前端做数据准备

---

## 延后内容

- 更多规则集
- 更多角色
- 向量检索 / RAG
- 联机大厅
- 前端应用

---

## 给开发 Agent 的执行建议

### 每个阶段都要做到：
- 有单元测试
- 有集成测试
- 有最小可运行演示
- 有明确退出条件

### 绝对不要：
- 一上来就接多家模型
- 规则层尚未稳定就开始写大量 Prompt
- 先写 UI 再补恢复与审计
