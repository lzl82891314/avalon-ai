# Avalon AI Java Maven 项目文档（中文版）

这套文档的目标不是直接提供完整代码，而是为 Codex / Claude Code 提供一套足够明确的工程约束、模块边界、类职责、任务拆分与落地顺序，使其可以基于这些 Markdown 文档直接构建一个可演进的 Java / Spring Boot / Spring AI / LangChain4j 项目。

适用目标：

- 第一版：纯 LLM 自动运行的阿瓦隆游戏
- 第二版：支持 Human / LLM / Scripted 混合玩家
- 第三版：支持断点恢复、回放、多人联机与前端 UI

- [02-模块职责与依赖边界.md](./02-模块职责与依赖边界.md)
- [03-核心接口与类设计.md](./03-核心接口与类设计.md)
- [04-配置与资源文件设计.md](./04-配置与资源文件设计.md)
- [05-Codex-ClaudeCode构建指令.md](./05-Codex-ClaudeCode构建指令.md)
- [06-最小落地顺序.md](./06-最小落地顺序.md)

建议 AI 编码代理执行顺序：

1. 先按 `01` 与 `02` 创建 Maven 多模块骨架
2. 再按 `03` 创建核心接口、实体、服务骨架
3. 按 `04` 增加规则配置、角色配置与应用配置
4. 按 `06` 的顺序分阶段实现最小功能
5. 最后根据 `05` 中的任务拆分逐模块补全实现
