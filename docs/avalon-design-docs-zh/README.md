# 阿瓦隆多 Agent 游戏平台设计文档（中文版）

本套文档面向 **Java / Spring Boot 后端项目**，目标不是做一个“几个 LLM 聊天玩桌游”的演示，而是构建一个**可持续演进、可审计、可恢复、可扩展**的阿瓦隆游戏平台后端。

文档风格采用 **A+B 混合模式**：
- **A（规范设计文档）**：用于架构评审、长期维护、团队对齐
- **B（AI 开发导向文档）**：用于直接交给 Codex / Claude Code 开始编码

---

## 一、项目目标

阶段目标：

- **V1：纯 LLM 自动对局**
  - 无需前端
  - 由后端自动创建、启动并跑完整局
  - 所有玩家由 LLM 控制
  - 每轮都可查看公开发言、结构化动作、审计摘要、核心状态

- **V2：单机人机模式**
  - 一个真人玩家替代一个 LLM
  - 轮到真人时暂停，等待输入
  - 其余玩家仍由 LLM 自动完成

- **V3：多人联机模式**
  - 多个真人玩家加入房间
  - LLM 可用于补位
  - 支持 SSE / WebSocket 实时事件推送

## 三、核心原则

- **规则引擎是唯一真相源**
- **LLM 是玩家，不是裁判**
- **每个玩家必须严格隔离视野与私有记忆**
- **每个关键动作必须可审计**
- **系统必须支持中断恢复与回放**
- **规则、角色、人数、控制器、模型都必须可扩展**
- **从第一天开始为未来前端与多人联机保留接口**

## 四、推荐技术栈

- Java 21
- Spring Boot 3.x
- Spring AI
- LangChain4j
- PostgreSQL
- Flyway 或 Liquibase
- Jackson
- Bean Validation
- Micrometer + OpenTelemetry
- SSE / WebSocket（后续实时推送）

---

## 五、推荐 Maven 模块

- `avalon-app`：Spring Boot 启动模块
- `avalon-core`：纯领域模型、规则引擎、状态机
- `avalon-config`：规则集 / 角色 / 开局模板配置加载与校验
- `avalon-agent`：LLM 接入、Prompt 构造、输出解析、记忆合并
- `avalon-runtime`：编排器、回合调度、控制器解析、运行时恢复
- `avalon-persistence`：事件流、快照、记忆快照、回放投影
- `avalon-api`：REST / SSE / WebSocket 接口
- `avalon-testkit`：脚本玩家、测试工装、夹具数据

## 六、V1 成功标准

- 能基于配置好的规则集创建一局 5 人阿瓦隆
- 能基于 seed 确定性分配身份
- 能运行一整局 5 个 LLM 的自动对局
- 能保存所有公开发言、结构化动作、审计摘要、阶段变更
- 能逐步回放整局
- 能在中断后依赖快照 + 玩家记忆继续运行

---

## 七、给 Codex / Claude Code 的使用建议

请将本套文档视为开发真相源，并遵守以下原则：

1. 先实现 `avalon-core`，不要先写 Prompt
2. 不要把规则写死在 LLM 提示词里
3. 不要把架构写死为只支持 5 人
4. 所有“影响游戏真相”的动作都必须持久化
5. 允许 LLM、真人、脚本三类控制器并存
6. 任何新增角色或规则，优先通过配置和元数据扩展，而不是 if/else 堆砌
