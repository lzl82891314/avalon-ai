# 06 - 最小落地顺序

## 1. 目标

本文件给出最务实的实现顺序。重点不是“一次做完所有特性”，而是让项目尽快从设计走向可运行。

---

## 2. 推荐阶段

### 阶段 A：只做骨架，不接 LLM

必须完成：

- Maven 多模块
- 核心领域模型
- 规则/角色/setup YAML 加载
- `RoleAssignmentService`
- `GameRuleEngine` 最小版本
- `ScriptedPlayerController`
- `GameOrchestrator`
- `run-to-end`

完成目标：

> 用脚本玩家跑完一局 5 人经典阿瓦隆。

这是第一里程碑。

---

### 阶段 B：加入事件与快照

必须完成：

- `GameEvent`
- `GameSnapshot`
- `PlayerMemoryState`
- 事件持久化
- 快照持久化
- 基础恢复能力

完成目标：

> 能查看整局事件流，并从中断点继续运行。

这是第二里程碑。

---

### 阶段 C：接入 LLM

必须完成：

- `ModelProfile`
- `ChatModelFactory`
- `AgentGateway`
- `LlmPlayerController`
- 结构化 JSON 输出解析
- per-player 模型绑定

完成目标：

> 5 个玩家可由不同模型扮演并自动跑完一局。

这是第三里程碑。

---

### 阶段 D：支持 HumanPlayerController

必须完成：

- 人类玩家动作等待机制
- API 提交动作
- `WAITING_HUMAN_INPUT` 状态

完成目标：

> 支持单机人机混合局。

这是第四里程碑。

---

## 3. 为什么这个顺序最稳

因为它满足三条原则：

1. 先把规则引擎做稳，再接 AI
2. 先把可恢复能力做出来，再谈长期演进
3. 先让系统可跑，再扩展 UI 和多人联机

如果反过来，一开始就上 UI、多人、复杂 agent，很容易失控。

---

## 4. 第一里程碑的验收标准

达到以下条件才算通过：

- Maven 多模块可编译
- 5 人经典局可创建
- 身份可随机分配
- 讨论 / 提名 / 投票 / 任务 / 刺杀完整流程可走通
- 不依赖任何 LLM
- 可以输出整局的文本日志

---

## 5. 第二里程碑的验收标准

达到以下条件才算通过：

- 每一步动作写入事件表
- 定期写入快照
- 中断后可重新加载局面
- 玩家记忆可以单独保存

---

## 6. 第三里程碑的验收标准

达到以下条件才算通过：

- 每个玩家可绑定独立模型
- 模型返回结构化 JSON
- 非法动作会被校验与兜底
- 可查看公开发言与审计摘要

---

## 7. 给 AI 编码代理的最后要求

请不要跳步骤。

正确顺序是：

1. 骨架
2. 规则
3. Scripted
4. 事件与恢复
5. LLM
6. Human

这样做出来的系统才会稳。
